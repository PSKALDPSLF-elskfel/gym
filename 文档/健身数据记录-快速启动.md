# 健身数据记录功能 - 快速启动指南

## 🚀 5分钟快速上手

### 第一步：确认后端服务运行 ✅

```bash
# 1. 确保MySQL服务运行
# 2. 确保数据库已初始化（执行了 workout_record_system.sql）
# 3. 启动Spring Boot应用
cd springboot
mvn spring-boot:run
```

**验证后端**: 访问 `http://localhost:8080`，应该看到服务正常运行。

---

### 第二步：配置小程序API地址 ✅

检查 `uniapp/config/site.js` 文件：
```javascript
export function getBaseUrl() {
  return 'http://localhost:8080'  // 确保指向正确的后端地址
}
```

---

### 第三步：启动小程序 ✅

1. 使用HBuilderX打开 `uniapp` 目录
2. 运行到浏览器或微信开发者工具
3. 登录测试账号

---

## 🎯 核心功能说明

### 1. 运动记录录入
**支持的运动类型**:
- 有氧运动：跑步、快走、游泳、骑行等
- 力量训练：深蹲、卧推、硬拉等
- 柔韧性训练：瑜伽、拉伸等
- 球类运动：篮球、足球等

**可记录数据**:
- 基本信息：类型、日期、时间、时长、强度
- 有氧数据：距离、步数、心率
- 感受信息：感受、天气、位置、备注

### 2. 运动历史查询
**功能特点**:
- 日期范围筛选
- 分页加载
- 分类标签
- 点击查看详情

### 3. 数据统计分析
**统计维度**:
- 总运动次数、时长、热量
- 连续运动天数
- 有氧/力量占比
- 时长趋势、热量趋势
- 本周/本月统计

## 🔍 常见问题快速解决

### Q1: 添加记录失败？
**检查清单**:
- [ ] 必填项是否都已填写
- [ ] 后端服务是否运行
- [ ] 是否已登录
- [ ] 网络连接是否正常

### Q2: 统计数据不显示？
**检查清单**:
- [ ] 是否有运动记录
- [ ] 日期范围是否包含记录
- [ ] 刷新页面重试

### Q3: 图表显示异常？
**解决方案**:
- 切换日期范围标签
- 检查是否有数据
- 查看控制台错误

## 🎓 使用技巧

### 技巧1：快速录入
- 日期默认为今天，通常无需修改
- 常见运动类型记住ID可快速选择
- 热量可以不填，后续可根据MET值自动计算

### 技巧2：数据筛选
- 使用日期范围可以查看特定时期的记录
- 近7天适合查看本周数据
- 近30天适合查看本月趋势

### 技巧3：数据分析
- 连续运动天数可以激励坚持
- 有氧/力量占比帮助平衡训练
- 趋势图可以看出进步情况
